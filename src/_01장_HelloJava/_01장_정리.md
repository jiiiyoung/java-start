## 자바는 **표준 스펙**과 **구현**으로 나뉜다.

### 자바의 표준 스펙
- 자바 컬파일러
- 자바 실행 라이브러리
- 자바 가상머신(JVM)

### 구현
- 오라클 Open JDK
- Adoptium Eclipse Temurin
- Amazon Corretto


[참고 페이지](https://whichjdk.com/ko/)


## 컴파일과 실행
### 자바의 컴파일과 실행단계
1. 개발자가 자바 코드를 작성
2. 자바 컴파일러를 사용해 소스코드를 바이트코드로 컴파일
   - 자바가 제공하는 javac 프로그램을 사용
3. 자바 프로그램 실행
   - 자바가 제공하는 java 프로그램 사용
   - 자바 가상 머신(JVM)이 실행되면서 프로그램 작동


## 응용 프로그램과 자바의 차이
- 윈도우 프로그램은 Mac, 리눅스와 같은 다른 운영체제에서 실행할 수 없다.
- 하지만 자바는 JVM이 있어 어떠한 환경이든 실행할 수 있다.

### JDK: 개발 도구 (컴파일러 포함)
- javac: 자바 컴파일러
- java: 실행 명령, JVM 실행(JVM: 바이트코드를 OS에 맞게 실행)
- jdb: 디버거
- javadoc: 문서생성기
- 기본 라이브러리
- JRE 전체 (JVM + 라이브러리) (≠ JAR)

### JVM 내부
- 클래스 로더
- 실행 메모리 구조
- 실행 엔진(JIT 포함)
